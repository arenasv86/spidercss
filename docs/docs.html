Introduction

Spider CSS is a powerful SASS and Flexbox framework focused on performance and flexibility for medium and large projects. It isn't another UI Framework like Bootstrap, it just provides common functions, helpers, mixins and some basic UI elements for build webapps.

Thanks to SASS you can configure the project with your own variables and generate helpers and tools that make easier the UI development and improve your workflow.

Let the game begin!

Installation

There are four simple ways to start working with SpiderCSS:

Install it via Bower 
$ bower install spidercss --save
Install it via npm 
$ npm install spidercss --save
Clone the repo 
$ git clone https://github.com/arenasv86/spidercss.git
Download from GithubDownload

Usage

There are two ways to getting started with Spider CSS:

SASS (Recomended)

I highly recommend to work directly with the source files in the project development. You'll can modify all the variables and configure it at your taste for build a strong project structure.

  1. Copy the dist folder to your project assets folder
  2. Import spidercss.scss at the top of your .scss main file
Compiled CSS

If you want, you can use the compiled stylesheet in your project. Just find the file spidercss.min.css in the project dist folder and include it inside your document head.

Basic Configuration

Okay, we have SpiderCSS installed in our project. Let's see some basic configuration for the project and how to adapt the framework to our needs.

In the source files, go to Settings > _setup.scss

Base Typograhpy 
$base-font-size: 16px !default;
$line-height-ratio: 1.5 !default;
$base-line-height: $base-font-size * $line-height-ratio !default;
Common Variables 
$base-radius: 4px !default;
$base-transition: .25s !default;
$outline-button-border-width: 2px;
Common Variables 
$generate-flexbox-grid: true; // Generate the grid classes. If you don't want to use the grid, set it to false.
$generate-flexbox-grid-offsets: false; //With flexbox you can align the columns within the row, so probably you'll don't need it. Anyway you can generate the offsets if you want.
$grid-columns: 12 !default; // The number of columns for the grid (the major part of grids uses 12 or 16)
$grid-columns-mobile: 4 !default; // The number of columns for the grid in mobile (If you want to use the 12 columns instead, change it.)
$gutter-width: 15px !default; // The spacing between grid columns.
$container-desktop-padding: 30px !default; //The horizontal padding for the container
$container-mobile-padding: 15px !default; //The horizontal padding for the container in mobile
$row-max-width: 1180px !default; // The max width size for the grid
